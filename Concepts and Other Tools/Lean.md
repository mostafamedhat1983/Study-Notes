- **Lean Principles**: Lean focuses on eliminating waste and maximizing value flow through a system. It has five principles: Identify value, map your value stream, create a flow, establish a pull system, and continuously improve.
- **Types of Waste**: Lean identifies three types of waste: Muda (non-value creating work), Mura (unevenness), and Muri (overburdening).
	**Muda**: Activities that consume resources but do not add value from the customer’s perspective.
	    
		**Type 1 Muda** : Non-value-adding work that may still be necessary under current conditions, such as compliance or regulatory steps.​
		    
		**Type 2 Muda** : Pure waste that adds no value and should be eliminated.​
	    
	- **Mura**: Unevenness, inconsistency, or irregularity in the workflow or process.
    
	- **Muri**: Overburden placed on people, equipment, or systems beyond reasonable capacity.
	
- **Value Stream Mapping**: This technique helps visualize where value is created and how it flows through the organization.
- **Continuous Improvement**: Lean emphasizes continuous improvement (Kaizen) by empowering individuals and teams to analyze and improve their work processes.

In Lean, pull means work starts because there is real downstream demand, while push means work is released based on forecasts, schedules, or assumptions. Lean prefers pull because it reduces waste such as overproduction, excess inventory, and long queues.

## Push vs pull

In a push system, a company produces items in advance and moves them forward based on a plan or forecast. In a pull system, production or replenishment is triggered only when the next step or the customer actually needs something.

## Why Lean prefers pull

Lean aims to deliver value with less waste, and overproduction is considered one of the most harmful forms of waste. Pull helps by limiting work in progress, reducing inventory, exposing bottlenecks and quality issues faster, and shortening lead times.

## Simple idea

Push says, “Make it now because we think it will be needed.” Pull says, “Make it now because it is needed now.” That shift is why pull is closely tied to just-in-time thinking in Lean.

## Example

Imagine a bakery. In a push system, it bakes 500 loaves in the morning based on expected demand, so some may remain unsold. In a pull system, it bakes or replenishes in response to actual sales, which better matches real demand and avoids waste.

## Important nuance

Lean does not always mean a pure pull system everywhere. Several sources note that many real operations use a hybrid model, where some upstream purchasing or planning is push-based, but production release or final configuration is pull-based.

In short, Lean uses pull rather than push because pull aligns work with actual demand and helps the whole system carry less waste.

Value stream mapping is a Lean method for visually showing every step needed to deliver a product or service, including both the material flow and the information flow. Its main purpose is to help teams see waste, delays, and non-value-added steps so they can improve the process.

## What it shows

A value stream map usually includes the key process steps, waiting time, cycle time, inventory or work in progress, and how information moves between people or systems. In Lean, it is used to understand the current state and then design a better future state.

Lean software development principles:  
  

- **Eliminate Waste**: Focus on reducing unfinished work, extra features, poor documentation, delays, context switching, and defects.
- **Amplify Learning**: Deliver working code in short cycles for immediate feedback.
- **Decide as Late as Possible**: Make decisions with maximum feedback.
- **Deliver as Fast as Possible**: Minimize the gap between customer needs and fulfillment.
- **Empower the Team**: Push decision-making down to individuals and educate them to improve their work.
- **Build Integrity In**: Find and eliminate bugs early in the process.
- **See the Whole**: Keep the entire system in mind when making decisions, using visualizations to stay focused on the same goal.

  
These principles help in optimizing the development process and improving software quality.
