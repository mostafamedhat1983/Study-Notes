MVP stands for **Minimum Viable Product**. It is the simplest version of a product that has just enough core features to deliver value, test the idea with real users, and collect feedback for future improvement.

## Simple meaning

An MVP is not a fake demo and not a full product. It is a **working** product with only the essential features needed to solve the main user problem.

## Why teams build it

Teams use an MVP to validate whether users actually want the product before spending too much time and money on extra features. This approach helps reduce risk, speed up learning, and guide future development using real feedback.

## Easy example

If you want to build a food delivery app, the MVP might only include signup, restaurant listing, order placement, and payment. Features like coupons, live driver tracking, loyalty points, and advanced analytics can come later.

## Good interview answer

You can say: “An MVP is the simplest functional version of a product with enough core features to provide value to early users and gather feedback for future development.”
